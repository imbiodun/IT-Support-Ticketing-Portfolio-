# IT Support Ticketing Portfolio

This repository demonstrates my hands-on experience with IT support, troubleshooting, and structured help-desk workflows.

Each ticket in this portfolio is based on real issues I have encountered on my personal system or while assisting others. These scenarios are documented using a professional help-desk format to reflect how issues are handled in a real IT support environment.

---

## Purpose

The goal of this portfolio is to demonstrate my ability to:

- Diagnose and resolve real-world technical issues
- Follow structured troubleshooting processes
- Identify root causes (not just apply quick fixes)
- Document issues clearly in a help-desk ticket format
- Communicate technical problems in a user-friendly way

---

##  What This Portfolio Shows

This portfolio reflects how I approach IT support tasks:

- **Problem Analysis:** Understanding user-reported issues and system symptoms  
- **Troubleshooting Workflow:** Step-by-step investigation and testing  
- **Root Cause Identification:** Determining the actual cause of the issue  
- **Resolution:** Applying effective fixes  
- **Documentation:** Recording actions clearly for future reference  

---

##  Technologies & Areas Covered

- Windows OS troubleshooting  
- Microsoft 365 (OneDrive, Outlook, Teams – user-level issues)  
- Networking (connectivity, DNS, basic configuration)  
- Hardware & system performance issues  
- Virtualization (VirtualBox environments)  

---

## 📂 Structure of the Repository

The tickets are organized into categories commonly found in IT support environments:

- `windows-issues` – OS errors, performance problems  
- `microsoft365` – OneDrive, Outlook, and cloud-related issues  
- `network-issues` – connectivity and configuration problems  
- `hardware` – device-related troubleshooting  

Each ticket includes:

- Issue Summary  
- Environment  
- User Description  
- Troubleshooting Steps  
- Root Cause  
- Resolution  
- Prevention Notes
